Build dependency — ONE signup page + UTM source capture [MARLA + MARK]

This is new for the team, so here's exactly what we're doing and why. Read this in full.

The setup: there is ONE VIP signup landing page. Both Meta pre-sale campaigns (the lead-optimised one and the purchase-optimised one) send people to that same page. We are NOT building two pages. The only place a signup happens differently is the Meta Instant Form (which is on-platform, no landing page).

The problem it solves: because both campaigns point at the same page, we need another way to know which campaign a signup — and later, a purchase — actually came from. Otherwise we can't work out new-customer cost per campaign.

How it works (the important part): every Meta ad link already carries a dynamic UTM that auto-fills from the campaign/ad names (see §4.5). When someone lands on the page and signs up, the Klaviyo form reads that UTM straight off the page URL and saves it onto their profile as a property — e.g. CIJ26 Signup Source = lead-opt or purch-opt. It must be read live from the URL, NOT a fixed value typed into the form (a fixed value can't tell the two campaigns apart).

Then, later, when that same person buys, Klaviyo already knows which campaign first signed them up (it's saved on their profile). So we can report new customers and revenue by originating campaign, and divide each campaign's spend by the buyers it produced.

Marla — your steps (Klaviyo):

  1. Build the VIP signup form in Klaviyo.
  2. Add a hidden field mapped to capture the URL parameter (utm_campaign, and optionally utm_content) and write it to a profile property called CIJ26 Signup Source.
  3. Test it: open the form using a link with ?utm_campaign=test123 on the end, submit, and check the profile now shows CIJ26 Signup Source = test123.

Mark — your steps (landing page):

  1. Embed Marla's Klaviyo form on the rebuilt LP.
  2. Make sure the page keeps the URL parameters intact through to the form — if the page strips ?utm_… off the URL, the capture won't work. (No redirect that drops the query string.)
  3. Test end-to-end with a real tagged link before pre-sale goes live.

Worked example: Jess names the lead-opt ad so its link becomes …/vip-access?utm_campaign=CIJ26-LEAD-OPT… → Sister clicks, signs up → her Klaviyo profile gets CIJ26 Signup Source = CIJ26-LEAD-OPT → two weeks later she buys → the sale report counts that purchase under the lead-opt campaign. Same logic for purchase-opt and the instant form.

Sequence: Marla builds + tests form → hands to Mark → Mark embeds + tests UTMs survive → both confirm end-to-end before pre-sale.
