5. Tracking — dynamic UTM template (set at AD level on EVERY pre-sale ad)

Per Master Brief §4.5 — UTMs auto-populate from the campaign/adset/ad names, so the work is correct naming, not hand-built UTMs.

Ad-level URL parameters (paste verbatim into the ad's "URL parameters" field):

utm_source={{site_source_name}}_{{placement}}&utm_medium=paidsocial&utm_campaign={{campaign.name}}&utm_id={{campaign.id}}&utm_content={{ad.name}}_{{ad.id}}&utm_term={{adset.name}}_{{adset.id}}
  • Because the names carry LEAD-OPT, PURCH-OPT, INSTANT-FORM, ATC, those source tags flow straight into reporting via utm_campaign / utm_term.
  • Instant-form leads (C1-B) bypass the LP, so their source is tagged at the form/integration level instead of via page UTM.

6. Klaviyo hidden-field source capture (the link to downstream measurement)

This is what lets us judge cost-per-acquired-customer per campaign (Master Brief §6, §4.6). Full build is in ../klaviyo/cij26-vip-list-spec.md. Meta-side requirements:

  1. Every Meta pre-sale ad's link carries the dynamic utm_campaign (e.g. resolves to …CIJ26 PRE-SALE LEAD-OPT…).
  2. The VIP LP must preserve the ?utm_… query string through to the embedded Klaviyo form (no redirect that strips it — Mark's job, Master Brief §6).
  3. The Klaviyo form reads utm_campaign (and optionally utm_content) live off the URL and writes it to profile property CIJ26 Signup Source — NOT a fixed value.
  4. When that profile later buys, Klaviyo already knows the originating campaign → we divide each campaign's spend by the buyers it produced.

Hard dependency: form + UTM capture tested end-to-end before pre-sale (Master Brief §6, §7 — due 15 Jun). Instant-form arm is tracked separately via the lead-ads integration.
