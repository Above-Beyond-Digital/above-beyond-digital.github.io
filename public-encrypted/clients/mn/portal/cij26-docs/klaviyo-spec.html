3. Hidden field — CIJ26 Signup Source from UTM (the measurement link)

This is what lets Meta judge cost-per-acquired-customer per campaign (Master Brief §6, §4.6). Read the full plain-English explainer in Master Brief §6.

How it works:

  1. Every Meta pre-sale ad link carries a dynamic utm_campaign that auto-fills from the campaign name (e.g. resolves to …CIJ26 PRE-SALE LEAD-OPT… / …PURCH-OPT…) — set at ad level (Meta pre-sale spec §5).
  2. The Klaviyo form has a hidden field mapped to read utm_campaign (and optionally utm_content) LIVE off the page URL and write it to profile property CIJ26 Signup Source.
  3. Must be read live from the URL — NOT a fixed value typed into the form (a fixed value can't tell the two campaigns apart). Master Brief §6.
  4. Later, when that profile buys, Klaviyo already knows the originating campaign → Meta reporting divides each campaign's spend by the buyers it produced.

Marla's steps (Master Brief §6):

  1. Build the VIP signup form in Klaviyo.
  2. Add the hidden field → capture utm_campaign (optionally utm_content) → write to profile property CIJ26 Signup Source.
  3. Test: open the form via a link with ?utm_campaign=test123, submit, confirm the profile shows CIJ26 Signup Source = test123.

Mark's steps (Master Brief §6):

  1. Embed Marla's Klaviyo form on the rebuilt LP.
  2. Ensure the page keeps the ?utm_… query string intact through to the form (no redirect that strips it).
  3. Test end-to-end with a real tagged link before pre-sale goes live.

Worked example (Master Brief §6): lead-opt ad link → …/vip-access?utm_campaign=CIJ26-LEAD-OPT… → Sister signs up → profile gets CIJ26 Signup Source = CIJ26-LEAD-OPT → she buys later → sale report counts that purchase under lead-opt. Same for purch-opt and the instant form.
